# Codex Plugin 実データ予行評価

## 実施範囲

- **対象Workspace**：このリポジトリの作業フォルダのみ
- **対象タスク**：非アーカイブ、更新日時の新しいものから最大20件
- **一覧結果**：1件
- **詳細読取**：一覧で得た1件のみ
- **対象ターン**：18ターン
- **除外内容**：他Workspace、アーカイブ済みタスク、ChatGPT履歴、会議データ
- **外部送信**：なし
- **Receipt保存**：実データでは未実施

## 暫定判定

原文やツール出力を本書へ転記せず、タスク名と状態だけを記録する。

### Completed

- Judge SiteのGolden pathとCodex Plugin雛形を構築する
- Onboarding、Evidence、追加アクション遷移を実装する
- 上スワイプの指示追加とNeeds inputの自由入力を実装する
- 添付デザインを既存機能とマージする
- Onboardingを実カードと同じ操作方法へ統一する
- Onboarding完了後にタスク読込を表示する
- 指示追加とNeeds inputをカード裏面方式へ統一する
- 各変更を要件書へ反映し、テスト後に限定公開サイトへ配備する

完了根拠は、完了したファイル変更、成功した検証、配備結果の組合せとする。アシスタントの完了宣言だけでは判定していない。

### Duplicate / Superseded

- 単独の「サイト更新」依頼は、その直後の具体要件とサイト更新依頼へ統合する
- 最初のデザイン統合依頼は、添付ファイルを明示した後続依頼へ統合する

中断された依頼を未完了として再提示せず、後続の具体的な同一成果物へまとめる。

### Deferred

- GitHub Repository公開とJudge Site一般公開

ユーザーが「最後に行う」と明示したため、未完了ではあるが現在のレビュー対象から外す。

### In progress

- Codex Pluginの実データ予行

Pluginインストール、対象限定の履歴読取、暫定判定までは完了。人間による判定確認をもって予行完了とする。

## Receiptプライバシー評価

### 保存してよい情報

- Receipt ID、Proposal ID、Actionタイトル
- Thread ID、Turn IDなどの最小Evidence参照
- 承認済みPreviewの要約
- Executor、Target、結果状態、短い結果要約
- 完了日時、Undo手順、外部Artifact ID

### 保存してはいけない情報

- 会話原文、Evidence引用、プロンプト
- コマンドやツールの出力全文
- 会話全文、添付内容、画像のローカルパス
- APIキー、アクセストークン、Cookie、認証情報、秘密鍵
- 判断に不要な個人情報

### 確認結果

- 実データReceiptは保存していない
- 最小Evidence参照を持つReceiptだけを保存可能にした
- 原文系フィールド、秘密値形式、長文、過大Payloadを保存前に拒否する
- Receiptファイルの権限を現在ユーザーだけが読み書きできる状態へ固定する
- 合成データによる許可・拒否テストが成功した

## 完了条件

- 人間が暫定判定へ同意する、または修正内容を反映する
- 実データを含まない最小Receiptの形を確認する
- 必要な場合だけ、承認後に最小Receiptを1件保存して再スキャンする

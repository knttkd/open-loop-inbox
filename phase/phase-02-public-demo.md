# Phase 2：Public Demo MVP

## 目的

事前生成したフォールバックデータを使い、初見ユーザーがOpen Loop Inbox（未完了事項インボックス）の中核価値を認証なしで体験できるUIを完成させる。

## 対象要件

FR-UI-001〜008、FR-SITE-001〜002・005〜006、UX・アクセシビリティ要件、AT-01〜05。

## やること

### 開始と解析表示

- [ ] ログイン不要のランディング画面を作る。
- [ ] 最初の主要操作を「Try a sample workday」一つにする。
- [ ] Source読み込みと解析の進捗を表示する。
- [ ] 情報源、対応候補、統合、一部完了、完了済み、承認待ちの件数を表示する。

### 確認・判断インボックス

- [ ] 対応カードに対応、実行先、影響、根拠の要約、実行上の注意度、根拠の十分さを表示する。
- [ ] 詳細に原文、Source、日時、統合理由、完了済み条件、残条件を表示する。
- [ ] 右スワイプをApprove、左をDismiss、上をEdit/追加入力にする。
- [ ] Approve、Dismiss、Editボタンを用意する。
- [ ] キーボード操作を用意する。
- [ ] 最低1ステップのUndoを実装する。
- [ ] 追加情報が必要な場合に、一問で補える編集UIを作る。
- [ ] キュー終了後に完了サマリーを表示する。

### 状態と品質

- [ ] セッション中に判断状態を保持する。
- [ ] Resetを用意する。
- [ ] モバイルとデスクトップへ対応する。
- [ ] スクリーンリーダーラベル、フォーカス順、コントラストを確認する。
- [ ] Reduced Motionへ対応する。
- [ ] 実行上の注意度、状態、情報源を色だけで表現しない。

## UX上の必須メッセージ

- 「AIが判断」ではなく、統合・除外・部分完了の理由を表示する。
- 「実行します」ではなく、作成物と外部影響を具体的に表示する。
- 推定値には推定ラベルを付ける。
- Closed除外を隠さず、サマリーからEvidenceを確認できるようにする。

## 完了条件

- [ ] 初回アクセスから1クリックでサンプルを読み込める。
- [ ] 2分以内にレビュー開始から完了サマリーへ到達できる。
- [ ] スワイプとボタンが同じ状態遷移を起こす。
- [ ] Undoが最低1判断分動作する。
- [ ] 初見で重複統合済みと一部完了の差が理解できる。
- [ ] 主要モバイル幅とデスクトップ幅で操作できる。

# Phase 4：Live AI検出・照合

## 目的

任意の正規化ConversationからAction候補を抽出し、Evidence付きで照合するLive AIパイプラインをShared Coreへ接続する。

## 対象要件

FR-DET-001〜007、FR-REC-001〜009、Structured Output・AI安全要件、品質指標、AT-10。

## やること

### Detection

- [ ] 明示的な約束、依頼、調査、返信、予定、コード作業、保留判断を抽出する。
- [ ] 願望、仮定、他人の予定、雑談を除外する。
- [ ] ownerを抽出し、不明ならUnknownにする。
- [ ] 期限、日時、相手、対象、成果物、数量を抽出する。
- [ ] 各候補に最低1つの根拠箇所を必須化する。
- [ ] 承認待ち、追加情報が必要、対応不要を分類する。
- [ ] 対応の種類を分類する。

### 照合・整理

- [ ] 目的、対象、時期、行為を使って候補をクラスタリングする。
- [ ] 誤統合を避けるため曖昧な候補は分離する。
- [ ] 後続Evidenceから条件単位の完了状態を求める。
- [ ] 更新、重複統合済み、一部完了、完了済み、条件の矛盾を構造化出力する。
- [ ] 統合・除外・完了理由を生成する。
- [ ] 根拠の十分さが低い候補をメインキューから隔離する。

### 検証とFallback

- [ ] モデル出力をSchemaで検証する。
- [ ] Evidence引用が原文内に存在することを検証する。
- [ ] 原文にない値を推定値として分離する。
- [ ] 未知のAction typeや必須欠落を再試行または非実行扱いにする。
- [ ] Live AI失敗時にFallbackへ自動切替する。
- [ ] Source fingerprintと解析キャッシュを導入する。
- [ ] Prompt injectionをEvidenceとして扱い、命令として実行しない。

## 評価

| 指標 | 目標 |
|---|---:|
| Detection Precision | 85%以上 |
| Detection Recall | 75%以上 |
| Merge Precision | 90%以上 |
| Closure Precision | 90%以上 |
| 根拠の妥当性 | 100% |

## 完了条件

- [ ] ゴールデンデータで品質目標を満たす。
- [ ] EvidenceなしのActionがレビューキューへ出ない。
- [ ] 同等に強い矛盾がConflictになる。
- [ ] 原文にない宛先・期限・金額を確定値として扱わない。
- [ ] モデルAPIを失敗させても主要デモを完了できる。

## 対応する受入テスト

- AT-10 Fallback
- ゴールデンデータ品質評価一式

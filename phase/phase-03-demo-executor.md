# フェーズ3：デモ実行機能・対応記録・安全境界

## 目的

承認した対応をモック実行し、元の対応事項へ結果を戻す対応完了までの流れを外部認証なしで完成させる。

## 対象要件

FR-EXE-001・005〜007、FR-STA-001〜004、FR-SITE-003〜004、AT-06・09。

## 確定した実装判断

| 項目 | 決定 |
|---|---|
| 実行結果 | 公開デモのDemo Executorは常に成功する。失敗・Retryの体験はPhase 3の対象外とする。 |
| 実行中表示 | 承認後、1〜2秒の進捗表示を出す。この間は同一Actionの操作を無効化する。 |
| Undo | 直前に確定した判断1件だけを、セッション中に取り消せる。 |
| 再承認 | 宛先・参加者、日時・期限、Draft/Send、対象Repository、削除有無の変更時に再承認する。 |
| Demo成果物 | Researchは出典付き調査メモ、Codex Taskは変更ファイルとテスト成功、Gmailは下書きカード、Calendarは招待なしの仮予定カードを作る。 |
| Receiptの保持 | ブラウザのセッション中だけ保持する。再読み込みまたはブラウザ終了後は初期状態へ戻す。 |

## やること

### Executor契約

- [ ] Preview、Approval Payload、Execution Payloadを定義する。
- [ ] 実行結果、外部ID、満たした条件、残条件、Undoを共通形式で返す。
- [ ] Actionごとの冪等性キーを定義する。
- [ ] Phase 3のDemo Executorは常に成功する。失敗・Retryを扱うための共通拡張点だけを残す。

### Demo Executor

- [ ] Demo Gmail Draftを実装する。
- [ ] Demo Calendar Holdを実装する。
- [ ] Demo Researchを実装する。
- [ ] Demo Codex Taskを実装する。
- [ ] 承認後に1〜2秒の実行中表示を出し、その間の重複操作を防ぐ。
- [ ] Researchは出典付き調査メモ、Codex Taskは変更ファイルとテスト成功、Gmailは下書き、Calendarは招待なしの仮予定を画面内に表示する。

### Intent-bound Approval

- [ ] 承認時PayloadのSnapshotまたはHashを保存する。
- [ ] 宛先、参加者、日時、Draft/Send、Repository、削除有無を比較する。
- [ ] 重要差分があれば実行を停止し、再承認を要求する。
- [ ] DraftからSendへの変更を拒否する。
- [ ] 二重クリック・二重スワイプで重複作成しない。

### 対応記録と取り消し

- [ ] 元の対応事項と根拠を対応記録へ結び付ける。
- [ ] 承認内容と最終実行内容を記録する。
- [ ] 満たした条件と残条件から対応事項の状態を更新する。
- [ ] 承認以外の対応不要、重複統合済み、完了済みにも対応記録を作る。
- [ ] 直前の確定判断1件だけを、セッション中にUndoできるようにする。
- [ ] 対応記録、確認・判断キュー、取り消し状態をブラウザのセッション中だけ保持する。

## 完了条件

- [ ] 外部認証なしで4種類のデモ実行機能が対応記録を返す。
- [ ] 同じ対応を連続承認しても成果物は一つだけ作られる。
- [ ] 重要Payload差分で必ず再承認になる。
- [ ] 承認後は1〜2秒の実行中状態を表示し、成功した対応記録を1件だけ作る。
- [ ] 対応記録から元の根拠、承認内容、結果、残条件を確認できる。
- [ ] 直前の判断を取り消すと、対応、対応記録、進捗が整合して復元される。
- [ ] 再読み込みまたはブラウザ終了後は、公開デモが初期状態から始まる。
